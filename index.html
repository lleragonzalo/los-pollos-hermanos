Welcome to the Landings Squad's
Fever Code Challenge

Before you start, make sure you have read these files carefully and in the following order:

  1. README.md
  2. PRD.pdf

They explain how this project works and what the test will consist of. As you navigate through the different files you will find two types of messages:

  • Informative messages that will inform you which areas of the project can be edited and which cannot. You can see them at the top of the files, if there is no message indicating it, it can be edited. (this does not mean that it should be done).

  • Code Hints that will give you useful advice during the development of your test.

Inside package.json you can find the only three commands you will need to do this test.

  • "dev" will open a development environment where you can see the changes you make in real time.

  • "build" will compile what you've built and generate a dist folder with your final web page.

  • "preview" will allow you to view that dist folder as if it were a real production environment.

When you submit your test, we'll use that last command to generate your static web page and test you, so it's important to make sure everything is working properly.

Good luck! 🙂
